6. DATA FLOW — END TO END
================================================================================

    Step 1: Sensing
    ────────────────
    Photoresistor ──ADC──▶ vEnvTask ──▶ xSensorQueue
    Big Sound ─────IRQ──▶ xSoundSemaphore
    HC-SR04 ───────GPIO──▶ vDistanceTask ──▶ xSensorQueue
    DHT11 ─────────GPIO──▶ ESP32 readDHT11()
    SW2/SW3 ───────IRQ──▶ vButtonTask

    Step 2: Processing (MCXC444)
    ─────────────────────────────
    xSensorQueue ──▶ vAlertTask
                         │
                         ├── Threshold checks (temp, humidity, light, noise, distance)
                         ├── Set warningLevel (0=Normal, 1=Warning, 2=Critical)
                         ├── Update globalSystemStatus (protected by xSystemStatusMutex)
                         └── Drive RGB LED (Green / Yellow / Red)

    Step 3: Communication (MCXC444 → ESP32)
    ─────────────────────────────────────────
    vCommsTask ──▶ Pack sensor data + warning level
               ──▶ UART TX: "TEMP:28.5,HUM:65.0,LIGHT:512,NOISE:1,DIST:45.2,WARN:1\n"

    Step 4: Cloud Operations (ESP32)
    ─────────────────────────────────
    uart_handler ──▶ parse into SensorData_t
         │
         ├──▶ firebase_client ──HTTPS──▶ Firebase RTDB
         │         PUT  https://<project>.firebaseio.com/latest.json
         │         POST https://<project>.firebaseio.com/readings.json
         │
         ├──▶ telegram_bot ──HTTPS──▶ Telegram Bot API
         │         GET  api.telegram.org/bot<token>/getUpdates
         │         POST api.telegram.org/bot<token>/sendMessage
         │
         └──▶ openai_client ──HTTPS──▶ OpenAI API
                   POST api.openai.com/v1/chat/completions
                   Model: gpt-4o-mini
                   Returns: one-sentence study advisory

    Step 5: Output (ESP32)
    ───────────────────────
    Advisory ──▶ LCD Display (scrolling text on line 2)
    Advisory ──▶ Telegram sendMessage (if critical)
    Advisory ──▶ UART TX back to MCXC444

    Step 6: Feedback Loop (ESP32 → MCXC444)
    ─────────────────────────────────────────
    ESP32 UART TX ──▶ vCommsTask UART RX
                          │
                          ├── "SETTEMP:30.0\n"  → update threshold in vAlertTask
                          ├── "SETDIST:40.0\n"  → update threshold in vAlertTask
                          └── "Advisory text\n"  → (optional display/log)

7. WARNING STATE LOGIC
================================================================================

    Condition                              Warning Level
    ─────────────────────────────────────  ─────────────
    All readings within thresholds         0 (Normal)    → RGB: GREEN
    Any ONE reading exceeds threshold      1 (Warning)   → RGB: YELLOW
    Multiple readings exceed OR            2 (Critical)  → RGB: RED
      distance critically close                           → Buzzer ON
                                                          → Telegram alert
                                                          → OpenAI advisory

    Default Thresholds:
      Temperature  > 32.0 C      → Warning
      Humidity     > 80.0 %      → Warning
      Light        < 200         → Warning (too dark)
      Noise        == 1          → Warning (noisy)
      Distance     < 30.0 cm     → Warning (too close)
